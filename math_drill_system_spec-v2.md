# 算数ドリル自動生成システム「スウロボ・ジェネレーター」開発仕様書 (v2)

## 1. システム概要
本システムは、ラオス語の学校教科書PDF（全18単元想定）を元に、AIエージェント（Antigravity）が内容を解析し、児童が楽しく学べる高品質な印刷用算数ドリル（学年別・1冊統合型）をWeb上に生成・デプロイするシステムである。

キャラクター「スウロボ」を活用した一貫性のあるUXデザイン、先生が授業で使いやすい紙面構成、そして解答・解説をドリル巻末に一括集約した製本に最適な構造を特徴とする。

### 1.1 対象言語
- **教科書・ドリル本文**: ラオス語（ພາສາລາວ）
- **フォント**: Phetsarath（ラオス語標準Unicode フォント）
- **数式・記号**: 言語非依存（アラビア数字・数学記号を使用）

---

## 2. アーキテクチャ & インフラ構成

### 2.1 コンテンツ生成方式
ドリルのコンテンツ（問題、解答、ヒント、SVG図解）は、**Antigravity（AIエージェント）が教科書PDFを直接解析し、HTML/SVG/CSSとして生成する**。自動化されたAI APIパイプラインではなく、Antigravityが開発環境内で直接ファイルを生成するワークフローを採用する。

```
教科書PDF（ラオス語） → Antigravity が解析 → HTML/SVG/CSS を直接生成 → Next.js SSG でビルド → Cloudflare Pages にデプロイ
```

### 2.2 インフラ構成（Cloudflare 無料プラン）

| コンポーネント | 採用技術 | 役割 |
| :--- | :--- | :--- |
| **フロントエンド** | **Next.js on Cloudflare Pages** | ドリルのWeb表示、印刷プレビュー、SSGによる静的サイト生成。 |
| **コンテンツ生成** | **Antigravity（AIエージェント）** | 教科書PDFの解析、問題・解答・ヒント・SVG図解の直接生成。 |
| **PDF出力** | **CSS Paged Media + ブラウザ印刷** | A4最適化された印刷用レイアウト。ブラウザの「PDFとして保存」で出力。 |
| **ホスティング** | **Cloudflare Pages（無料プラン）** | 静的サイトとしてデプロイ。カスタムドメイン対応。 |
| **フォント** | **Phetsarath** | ラオス語テキスト表示用の標準フォント。 |

> **注記**: Cloudflare 無料プランのため、Browser Rendering（サーバーサイドPDF生成）は使用しない。代わりに CSS `@media print` と `@page` ルールで印刷最適化を行い、ブラウザの「PDFとして保存」機能で高品質なA4 PDFを出力する。

## 3. 画面・紙面レイアウト仕様（印刷最適化）
ドリルは「Webで閲覧でき、そのままA4用紙に最適化されて印刷できる」ことを前提に、CSS Paged Mediaを用いて厳密にレイアウトを制御する。

### 3.1 共通ヘッダー構造
先生が生徒に対して授業中に指示を出しやすく、児童の迷子を防ぐための直感的なデザインを採用。
- **巨大なページ番号（左上）**: 遠くからでも見やすい太字の大フォントで配置（例: `font-size: 48pt;`）。
- **単元名と学習内容（右上）**: ページ番号の右側に、現在の単元タイトルと「何を学ぶか」の目標を明記（ラオス語表記）。
- **教科書対応表示**: 「ປຶ້ມແບບຮຽນ ໜ້າ 24〜27」のように、元になったPDFの該当ページ数を小さく併記。

### 3.2 学習管理・採点エリア
- **日付記入欄**: 全ページ上部に配置。学習した日を記録できるようにする（ラオス語ラベル）。
- **得点記入欄**: 各問題ページ右上（または右下）に「 $\quad$ / 100 ຄະແນນ」の採点枠を設置。達成感を視覚化し、先生や保護者の丸付けをスムーズにする。

### 3.3 スウロボ君のヒントボックス
- **配置条件**: 全てのページではなく、解き方のつまずきや応用問題の近くにAntigravityが動的に配置。
- **視覚デザイン**: パステルカラー（ライトグリーンや薄いブルー）の背景に左ボーダーをあしらったボックス。
- **キャラクター演出**: ボックス内に「スウロボ君」のSVGイラスト（思考中・ひらめき等の表情）と、吹き出し形式のテキスト（ラオス語）を配置。

## 4. キャラクターUI/UX＆世界観（スウロボ）
### 4.1 世界観設定
- **名称**: スウロボ (SUU-ROBO)
- **正体**: 宇宙の「すうじの国」からやってきた算数ナビゲーターロボット。
- **性格**: 計算が大好きで、新しい問題に出会うと胸のハート（Sマーク）が光る。間違えることを「新しい発見のチャンス」と捉え、児童を優しく肯定する。
- **デザイン特徴**: 柔らかいマットホワイトとブルーのボディ、頭部の2本のアンテナ、ダークバイザーにシアン色の表情ライト、胸のオレンジ色ハートSマーク。
- **ベースデザイン**: `SuuRobo.png` を基準とするSVGコンポーネント群。

### 4.2 ページコンテキストに応じたSVG配置ルール
Antigravityが生成する全てのHTMLページには、役割に応じたスウロボのSVGコード（拡大印刷しても絶対にぼやけない、正確な線画・フラットデザイン）を必ず埋め込むこと。

- **総合表紙**: 両手を広げた元気に挨拶するポーズ。
- **要点ページ**: 指し棒やペンを持って、黒板を背に解説するポーズ。
- **基礎問題**: ガッツポーズや、児童を応援するポーズ。
- **応用問題（ヒント内）**: 手を顎に当てて「うーん」と一緒に考えているポーズ。
- **巻末解答シート**: 丸メガネをかけた博士風のポーズ（丸付けや見直しの楽しさを演出）。
- **修了証**: クラッカーを持って全力でお祝いしているポーズ。

## 5. ドリル構成およびページネーション仕様（学年別・1冊統合型）
1学年全18単元を1冊のドリルとして美しく製本できるよう、以下の構造でHTMLを1本化して出力する。CSSの `page-break-after: always;` を用いて改ページを厳密に制御する。単元内の問題ページは3ページ構成にスリム化し、解答・解説はすべてドリル後半（巻末）に一括集約する。

### 5.1 フロントパート（P1 〜 P2）
- **P1: 総合表紙**
  - 学年タイトル（ラオス語。例：「ສູໂຣໂບ ກັບ ການຜະຈົນໄພ！ ຄະນິດສາດ ປ.2」）
  - クラス・名前記入欄（ラオス語ラベル）、スウロボのメイン挨拶SVG。
- **P2: 冒険の地図（目次兼進捗シート）**
  - 全18単元がRPGの「ステージ」のように双六（すごろく）状に並んだデザイン。
  - 1単元クリアするごとに、児童が色を塗ったりシールを貼ったりできる進捗管理シート。

### 5.2 メイン問題パート（P3 〜 P56）※3ページ × 18単元のループ構造
各単元の問題パート。すぐに答えが目に入らないよう、問題のみを連続して配置する。

- **単元内 P1: スウロボの要点チェック（インプット）**
  - 教科書PDFから抽出した公式、定義、グラフの読み方の図解（SVG）。
  - スウロボによる分かりやすい導入レクチャー（ラオス語）。
- **単元内 P2: きほん問題（ステップ1）**
  - 基本的な計算や、シンプルな図形・グラフの読み取り問題を3〜4問配置。
  - 視認性の高い巨大な「単元ページ番号」をヘッダーに配置。
- **単元内 P3: チャレンジ問題（ステップ2）**
  - 思考力を問う文章題やひねった応用問題を2〜3問配置。
  - 適所に「スウロボ君のヒントボックス」を配置。ページ下部には保護者・先生のハナマル用スタンプ枠を配置。

### 5.3 バックパート：巻末一括解答 ＆ 修了証（P57 〜 ）
問題パートがすべて終了したあと、ドリルの巻末に全単元の解答・解説を順番にまとめて掲載する。

- **P57 〜 : 巻末「回答シート」と「解説」エリア**
  - 18単元の解答を単元順にレイアウト。児童が自分で丸付け（自己採点）しやすく、先生や保護者も一気に答え合わせができる一覧性の高いデザイン。
  - 縮小された問題図（赤字での解答プロット付き）や、計算の途中式を美しく並べる。
  - スウロボ博士からの「間違えやすいポイントのワンポイントアドバイス」や解説SVGを適所に配置。
- **最終前ページ: 年間総まとめ（ファイナルボス・ミッション）**
  - 1年間の総仕上げとなる総合テストページ。
- **最終ページ: 年間修了証（賞状）**
  - 1冊やりきった児童の名前、日付を記入できる豪華な賞状デザイン。スウロボがクラッカーを持って全力でお祝いしているSVGを配置。

## 6. コンテンツ生成ルール（Antigravityへのインプット）
Antigravityがドリルを生成する際、画像生成モデル（DALL-E等）で数式や図形を描画することは**厳禁**とする（文字バグ、数字の歪みが発生するため）。

### 6.1 グラフィック生成の3原則
1. **すべてコード（SVG）で出力すること**: 三角形、円、棒グラフ、折れ線グラフ、キャラクターイラストは、すべてクリーンで軽量なHTML/SVGコードとして出力する。
2. **数理的正確性の担保**: グラフの目盛りや図形の比率は、問題の数値（例: 底辺4cm、高さ3cm）と完全に一致するSVG座標を計算して出力する。
3. **印刷対応カラー**: モノクロ印刷、カラー印刷のどちらでも視認性が落ちないよう、高コントラストなカラーシステム（ベース：Cloudflareオレンジ `#F38020`、アクセント：教育ブルー `#0051AD`）に準拠させる。

### 6.2 生成ワークフロー
```
1. ユーザーが教科書PDFをプロジェクトフォルダに配置
2. Antigravity が PDF を解析（単元構造、公式、練習問題を抽出）
3. Antigravity が単元データファイル (TypeScript) を生成
4. Antigravity が各単元の問題・解答データを生成
5. Antigravity が SVG 図解（グラフ、図形）を生成
6. Next.js SSG でビルド → Cloudflare Pages にデプロイ
7. ユーザーがプレビュー確認 → フィードバック → Antigravity が修正
```
