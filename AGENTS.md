# ⚠️ 過去の修正実績の上書き防止ルール（デグレード防止）

- **スクリプト・自動生成実行前のGit確認義務**:
  大問の拡充やデータの追加など、ファイル全体に影響するスクリプトを実行したり、サブエージェントに作業を依頼する前には、**必ず現在のワーキングツリーの変更をコミット（または `git diff` で差分を完全に点検）し、ユーザーがこれまでに指導したデザイン（文字サイズ、アドバイスボックスの有無、筆算スタイル等）が確実にファイルに含まれていることを点検しなければならない。**
- **「デザインの先祖返り」の禁止**:
  ファイルを上書き・一括生成する際、過去のメッセージや指導テキストを必ずスキャンし、過去にユーザーが決定したデザイン仕様（例: 「文字サイズ18px」「筆算はフランス式で小数点は赤」など）が、新しい自動生成データによって古い状態（先祖返り）に上書きされないよう、生成スクリプト側に手動の修正内容を事前にマージするか、変更後にコードの差分をセルフレビューして保護しなければならない。

- **自律的連続実装（ノンストップ実行）**:
  ビルドやテストが成功・終了したログが出た場合、そこで立ち止まってユーザーに確認や報告のためのチャットメッセージを送信して待機するのではなく、それをトリガーにして即座に次の単元の実装（ファイル作成・書き換え）に自律的に移らなければならない。特に、ビルドが成功・終了したログが出ても立ち止まらず、それをトリガーにして即座に次の単元の実装（ファイル書き換え）に移ること。
